# Reverb（混响）

- [English](./README_REVERB.md)

`Reverb`（混响）模块用于模拟声音在封闭空间中自然反射的效果。本实现采用 Freeverb（Schroeder-Moorer）算法，由 8 个并行梳状滤波器（带低通阻尼反馈）和 4 个串联全通滤波器（用于扩散）组成，算法轻量化，适用于 ESP32 嵌入式平台。

## 特性

- 支持全采样率与声道配置
- 支持位深：s16、s24、s32
- 支持数据布局：交织（interleaved）与非交织（non-interleaved）
- 可调参数：
  - `room_size`：房间大小因子，控制梳状滤波器反馈增益与混响尾部长度，范围 [0.0, 1.0]
  - `damping`：高频阻尼因子，控制反馈环路中高频吸收速率，范围 [0.0, 1.0]；值越大高频衰减越快
  - `wet_level`：湿信号（混响）电平，范围 [-96.0, 0.0] dB
  - `dry_level`：干信号（原始）电平，范围 [-96.0, 0.0] dB
  - `pre_delay_ms`：混响前置延迟时间，在混响开始前增加初始延迟，范围 [0, 200] ms（仅在创建时通过配置设置）

## 性能

测试环境：<br>
| 芯片 | IDF 版本 | CPU 频率 | SPI RAM 频率 |
| -- | -- | -- | -- |
| ESP32-S3R8 | v5.3 | 240MHz | 80MHz |

16 kHz 单声道：

| 位深 | 堆内存（Byte） | CPU 占用（%） |
| -- | -- | -- |
| 16 | < 20k | < 2.3 |
| 24 | < 20k | < 2.4 |
| 32 | < 20k | < 2.3 |

48 kHz 单声道：

| 位深 | 堆内存（Byte） | CPU 占用（%） |
| -- | -- | -- |
| 16 | < 59k | < 6.8 |
| 24 | < 59k | < 7.0 |
| 32 | < 59k | < 6.9 |

说明：
1) 内存占用主要由梳状滤波器和全通滤波器的延迟线决定，与采样率成正比，与声道数成正比。双声道内存约为单声道的 2 倍。
2) 不同采样率与声道配置的 CPU 占用可按如下公式估算：
   > CPU Load ≈ (sample_rate / 16000) × channel_count × base_load<br>

   其中 `base_load` 为 16 kHz 单声道表中给出的基准占用。
3) 表格中的 CPU 占用为平均测量值。

## 常见问题（FAQ）

1) 混响内存占用过大怎么办？
   > 内存主要被延迟线占用，与采样率成正比。降低采样率（如从 48 kHz 降至 16 kHz）可显著减少内存占用（约 3 倍）。

2) `room_size` 和 `damping` 如何配合使用？
   > `room_size` 越大混响尾部越长，`damping` 越大高频衰减越快。大 `room_size` + 低 `damping` 可模拟大厅效果；小 `room_size` + 高 `damping` 可模拟小房间效果。

3) `pre_delay_ms` 的作用是什么？
   > 前置延迟模拟声源到反射面的距离，增加空间感。一般设置 10-40 ms 即可获得自然的空间效果。该参数仅在 `esp_ae_reverb_open()` 时通过配置结构体设置，创建后不可动态修改。
